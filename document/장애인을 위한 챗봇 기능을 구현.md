# 장애인을 위한 챗봇 기능을 구현하는데 있어서, 우리 프로젝트가 활용할 수 있는 고용노동부와 한국산업인력공단, 안전보건공단, 근로복지공단, 한국고용정보원, 한국장애인고용공단, 한국사회적기업진흥원의 공공데이터를 가져오는 방법을 하나씩 세세하게 알려줘

장애인 취업 플랫폼의 AI 챗봇 개발을 위해 다양한 공공기관의 데이터를 활용하는 방법을 안내해드리겠습니다. 각 기관별로 공공데이터를 가져오는 방법을 세부적으로 설명해드리겠습니다.

## 공공데이터 포털을 통한 기본 접근 방법

공공데이터를 활용하기 위한 일반적인 절차는 다음과 같습니다:

1. 공공데이터 포털(www.data.go.kr) 회원가입 및 로그인
2. 원하는 데이터 검색 및 선택
3. 해당 데이터의 활용신청
4. 개발계정 발급 및 API 키 확인
5. 제공되는 API 가이드에 따라 데이터 활용

## 고용노동부 공공데이터

고용노동부의 공공데이터는 다음과 같은 방법으로 접근할 수 있습니다:

1. **접근 경로**:
    - 고용노동부 누리집(https://www.moel.go.kr) 접속 후 '정보공개 > 공공데이터개방' 메뉴 선택[^1_1][^1_16]
    - 또는 공공데이터 포털에서 '고용노동부' 검색
2. **제공 데이터 예시**:
    - 일자리/고용 분야: 구인구직현황, 중장년일자리 사업 통계, 강소기업 선정현황
    - 노사관계/근로기준 분야: 노동조합조직현황, 연도별최저임금, 퇴직연금도입현황
    - 산업안전보건 분야: 산업재해현황, 산업재해보상금액기준
    - 직업능력개발 분야: 직업능력개발 사업현황, 인적자원개발 우수기관 인증 현황[^1_16]
3. **활용 방법**:
    - 공공데이터 포털에서 발급받은 인증키(API 키)를 이용하여 REST API 호출
    - 제공되는 파일데이터의 경우 직접 다운로드하여 활용

## 한국산업인력공단 공공데이터

한국산업인력공단의 데이터는 다음과 같이 활용할 수 있습니다:

1. **접근 방법**:
    - 공공데이터 포털에서 '한국산업인력공단' 검색
    - 오픈 API 탭에서 원하는 데이터 선택 후 활용신청[^1_2]
2. **주요 제공 데이터**:
    - 직업방송 오픈 API 등록 및 사용정보 서비스[^1_17]
    - HRDK 한국산업인력공단 API 서비스
3. **활용 절차**:
    - 공공데이터 포털 로그인 → 데이터찾기 → 데이터목록(오픈 API 탭) → 해당 데이터 선택 → 활용신청[^1_2]
    - 발급된 서비스 키는 마이페이지 > 오픈 API 메뉴에서 확인 가능[^1_2]
    - API 활용 비용은 대부분 무료이나, 서비스별 상세정보 확인 필요[^1_2]
4. **API 호출 예시** (직업방송 API의 경우):
```python
import requests

url = "API_ENDPOINT_URL"
params = {
    'serviceKey': '발급받은_인증키',
    'openapiKey': 'OPEN_API_키',
    'stDt': '20250505'  # 조회 대상 일자
}
response = requests.get(url, params=params)
```


## 한국산업안전보건공단 공공데이터

한국산업안전보건공단의 데이터 활용 방법은 다음과 같습니다:

1. **접근 방법**:
    - 공공데이터 포털에서 '한국산업안전보건공단' 검색
    - 제공되는 API 목록에서 필요한 데이터 선택
2. **주요 제공 데이터**:
    - 안전보건법령 스마트검색 API[^1_3]
    - 국내재해사례 게시판 정보 조회서비스[^1_12]
3. **활용 방법**:
    - REST API 형식으로 제공되며, JSON 형태로 데이터 응답
    - 개발계정의 경우 일반적으로 10,000건의 트래픽 제공
    - 운영계정은 활용사례 등록 시 트래픽 증가 가능[^1_3][^1_12]
4. **API 호출 예시** (국내재해사례 조회):
```python
import requests
import json

url = "API_ENDPOINT_URL"
headers = {'Content-Type': 'application/json'}
params = {
    'serviceKey': '발급받은_인증키',
    'pageNo': '1',
    'numOfRows': '10'
}
response = requests.get(url, headers=headers, params=params)
data = json.loads(response.text)
```

    - 제공되는 API 목록에서 필요한 데이터 선택
2. **주요 제공 데이터**:
    - 근로복지지원현황 API[^1_4]
    - 고용/산재보험 현황정보 API[^1_13]
3. **활용 방법**:
    - REST API 형식으로 제공되며, XML 형태로 데이터 응답
    - 개발계정의 경우 일반적으로 10,000건의 트래픽 제공
    - 심의유형은 개발단계와 운영단계 모두 자동승인으로 설정됨[^1_4][^1_13]

```python
import requests
import xml.etree.ElementTree as ET

url = "API_ENDPOINT_URL"
params = {
    'serviceKey': '발급받은_인증키',
    'pageNo': '1',
    'numOfRows': '10',
    'v_saeopjaDrno': '사업자등록번호',  # 선택적
    'opaBoheomFg': '1'  # 1: 산재, 2: 고용
}
response = requests.get(url, params=params)
root = ET.fromstring(response.content)
```


## 한국고용정보원 공공데이터

한국고용정보원의 데이터는 다음과 같이 활용할 수 있습니다:

1. **접근 방법**:
    - 공공데이터 포털에서 '한국고용정보원' 검색
    - 또는 워크넷 OpenAPI 사이트(http://openapi.work.go.kr) 직접 접속[^1_14]
2. **주요 제공 데이터**:
    - 워크넷 직무데이터사전 API[^1_5]
    - 채용 정보 API, 직업 정보 API, 학과 정보 API, 정부지원 일자리정보 API[^1_14]
3. **활용 방법**:
    - 워크넷 OpenAPI 사이트에 회원가입 후 인증키 신청
    - 발급된 인증키는 서비스 이용허가를 받은 기관을 식별할 수 있는 고유키로, 타 기관에 양도 불가[^1_14]
    - API 유형은 LINK 방식으로 제공되며, XML 형태로 데이터 응답[^1_5]
4. **API 호출 예시**:
```python
import requests

url = "https://www.work24.go.kr/cm/e/a/0110/selectOpenApiSvcInfo.do"
params = {
    'apiSvcId': '000000000000000000000000000085',
    'upprApiSvcId': '000000000000000000000000000084',
    'fullApiSvcId': '000000000000000000000000000096^000000000000000000000000000086',
    'serviceKey': '발급받은_인증키'
}
response = requests.get(url, params=params)
```

한국사회적기업진흥원의 데이터는 다음과 같이 활용할 수 있습니다:

    - 공공데이터 포털에서 '한국사회적기업진흥원' 검색
    - 제공되는 데이터 목록에서 필요한 데이터 선택
2. **주요 제공 데이터**:
    - 사회적기업 상품 현황 파일데이터[^1_9]
3. **활용 방법**:
    - CSV 형식의 파일데이터로 제공되며, 로그인 없이 직접 다운로드 가능
    - 또는 공공데이터활용지원센터에서 제공하는 오픈 API(JSON/XML)로 자동변환된 데이터 활용 가능[^1_9]

```python
import pandas as pd

# CSV 파일 다운로드 후 로드
df = pd.read_csv('사회적기업_상품_현황.csv', encoding='utf-8')

# 데이터 분석 및 처리
```

## 공통 구현 방법

위의 모든 공공데이터를 통합하여 장애인 취업 플랫폼 AI 챗봇에 활용하기 위한 공통 구현 방법은 다음과 같습니다:

1. **데이터 수집 및 통합**:
    - 각 기관의 API를 호출하여 데이터 수집
    - 파일데이터의 경우 주기적으로 다운로드하여 데이터베이스에 저장
    - 데이터 형식 통일 및 정규화
2. **데이터 저장**:
    - 수집된 데이터를 MongoDB나 PostgreSQL 등의 데이터베이스에 저장
    - 정기적인 데이터 업데이트 스케줄링
3. **API 개발**:
    - FastAPI나 Spring Boot를 활용하여 내부 API 개발
    - 챗봇이 필요한 데이터를 쉽게 조회할 수 있는 엔드포인트 구성
4. **챗봇 연동**:
    - 사용자 질문에서 키워드 추출
    - 추출된 키워드를 기반으로 적절한 데이터 API 호출
    - 결과를 사용자 친화적인 형태로 가공하여 응답

이러한 방법으로 다양한 공공기관의 데이터를 활용하여 장애인 취업 관련 정보를 제공하는 AI 챗봇을 효과적으로 구현할 수 있습니다.

```python
from redis import Redis
from pymongo import MongoClient

class HybridCache:
    def __init__(self):
        self.redis = Redis()
        self.mongo = MongoClient().cache_db
        
    def get(self, key):
        if (cached := self.redis.get(key)):
            return cached
        else:
            data = self.mongo.find_one({'_id': key})
            self.redis.setex(key, 3600, data)
            return data[^2_5][^2_7][^2_15]
```
